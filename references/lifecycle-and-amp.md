# Page Lifecycle, Removals & AMP
Reference for site lifecycle operations — A/B testing, pausing a business, site moves, removals/redaction, image blocking, outbound-link qualification, canonicalization fixes, and AMP setup/removal. Grounded only in the local Search Central fork; every claim is an exact quote + repo-relative source path. Redirect-type semantics (301/308 permanent vs 302/303/307 temporary, JS-last-resort, meta-refresh) are NOT restated here — see `references/indexing-canonical.md`.

## Hard requirements

- **[A/B]** Test redirects must be temporary, not permanent: "If you're running a test that redirects users from the original URL to a variation URL, use a `302 (temporary)` redirect, not a `301 (permanent)` redirect." — `docs/search/docs/crawling-indexing/website-testing.md`
- **[A/B]** No cloaking during tests: "Don't show one set of URLs to Googlebot, and a different set to humans. This is called cloaking, and is against our spam policies, whether you're running a test or not." — `docs/search/docs/crawling-indexing/website-testing.md`
- **[A/B]** Cloaking is method-agnostic: "Cloaking counts whether you do it by server logic or by robots.txt, or any other method." — `docs/search/docs/crawling-indexing/website-testing.md`
- **[Pause]** Short outage → `503`: "If you need to urgently disable the site for 1-2 days, then return an informational error page with a `503` HTTP response status code instead of all content." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** Longer outage → indexable `200` placeholder: "If you need to disable the site for a longer time, then provide an indexable home page as a placeholder for users to find in Search by using the `200` HTTP status code." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** robots.txt must stay crawlable: "Continue to allow crawling through the robots.txt file. Don't return a `503` HTTP response status code for the robots.txt file because this blocks all crawling." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** (what NOT to do) "Don't disallow all crawling in the robots.txt file. Returning a valid robots.txt file that disallows all crawling may remove the website's content, and potentially its URLs, from Google Search." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** (what NOT to do) "Don't block the website by returning `403`, `404`, `410` HTTP status codes, or with a `noindex` robots `meta` tag or x-robots-tag HTTP header. This will remove the website's URLs from Google Search." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** (what NOT to do) "Don't use the temporary website removal tool in Search Console for closures. Doing so will make it impossible for users to find your website so that they can learn its status." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** (what NOT to do — robots.txt caveat) "Don't block your robots.txt file with a `503` HTTP response status code." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Removals]** Do NOT block with robots.txt: "Don't use robots.txt as a way to block your page." — `docs/search/docs/crawling-indexing/remove-information.md`
- **[Removals]** Correct permanent mechanism is one of: "Remove or update the content on your page." / "Password-protect your page." / "Add a `noindex` tag to your page." — `docs/search/docs/crawling-indexing/remove-information.md`
- **[Images]** X-Robots-Tag `noindex` requires the image URL to stay crawlable: "you can remove images hosted on your site from Google's search results by adding the `noindex` `X-Robots-Tag` to the HTTP response headers of the images you want to remove. In this case you must allow crawling the image URLs in order for Googlebot to be able to extract the `noindex` rule." — `docs/search/docs/crawling-indexing/prevent-images-on-your-page.md`
- **[Redacted]** Strip data before publishing: "it's critical that the information is removed completely from the file before that file is made public." — `docs/search/docs/crawling-indexing/keep-redacted-information-out.md`
- **[AMP]** An AMP page must link to a canonical: "For crawling and indexing, Google Search requires that an AMP page links to a canonical page. The canonical page can be either a non-AMP version of the page or it can be the AMP page itself." — `docs/search/docs/crawling-indexing/amp/enhance-amp.md`
- **[AMP]** rel pairing (both directions): "Did you add `rel="amphtml"` to the canonical page?" and "Did you add `rel="canonical"` to the AMP page?" — `docs/search/docs/crawling-indexing/amp/validate-amp.md`
- **[AMP]** Page must be valid: "Your AMP page must be valid so that your pages work as expected for users." — `docs/search/docs/crawling-indexing/amp.md`
- **[AMP]** Content/action parity with canonical: "Users must be able to experience the same content and complete the same actions on AMP pages as on the corresponding canonical pages, where possible." — `docs/search/docs/crawling-indexing/amp.md`
- **[AMP]** Don't empty the file to remove an AMP page: "Don't remove AMP content by deleting content in the file. Documents that are empty and missing all markup are considered invalid." — `docs/search/docs/crawling-indexing/amp/remove-amp.md`

## Recommendations

- **[Pause]** Prefer limiting functionality over removal: "we recommend that you keep your site online and limit the functionality." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** Full disable is extreme: "This is an extreme measure that should only be taken for a very short period of time (a few days at most), as it will otherwise have significant effects on the website in Search, even when implemented properly." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** Keep the status banner out of snippets: "To prevent the content in the banner or popup from being shown in a snippet in Search results, use the `data-nosnippet` HTML attribute." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[Pause]** Don't remove out-of-stock products via the Removals tool: "No. If you do this, customers won't be able to find first-hand information about your products on Search ... It's better to still allow that page, and mark it out of stock." — `docs/search/docs/crawling-indexing/pause-online-business.md`
- **[A/B]** Prefer `rel="canonical"` over `noindex` on variations: "We recommend using `rel="canonical"` rather than a `noindex` `meta` tag because it more closely matches your intent in this situation." — `docs/search/docs/crawling-indexing/website-testing.md`
- **[A/B]** JS redirects are acceptable for tests: "JavaScript-based redirects are also fine." — `docs/search/docs/crawling-indexing/website-testing.md`
- **[A/B]** Don't over-run experiments: "If we discover a site running an experiment for an unnecessarily long time, we may interpret this as an attempt to deceive search engines and take action accordingly." — `docs/search/docs/crawling-indexing/website-testing.md`
- **[Site move / URL change]** Recommended redirect codes: "we recommend that you use HTTP permanent redirects if possible, such as `301` and `308`." — `docs/search/docs/crawling-indexing/site-move-with-url-changes.md`
- **[Site move / URL change]** Use server-side permanent redirects old→new: "We recommend server side permanent redirects from the old URLs to the new URLs as you indicated in your mapping." — `docs/search/docs/crawling-indexing/site-move-with-url-changes.md`
- **[Site move / URL change]** Client-side redirects only as fallback: "If none of the server side redirect setups are possible, you can fall back to client side redirects as a last resort." — `docs/search/docs/crawling-indexing/site-move-with-url-changes.md`
- **[Site move / URL change]** Avoid redirect chains: "we advise redirecting to the final destination directly. If this is not possible, keep the number of redirects in the chain low, ideally no more than 3 and fewer than 5." — `docs/search/docs/crawling-indexing/site-move-with-url-changes.md`
- **[Site move / no URL change]** Scope: "This guide is only for migrations that don't affect the user-visible URL." — `docs/search/docs/crawling-indexing/site-move-no-url-changes.md`
- **[Site move / no URL change]** `noindex` the test hostname: "To prevent accidentally letting the test site get indexed, add the `noindex` `robots` rule to the HTML or the HTTP headers of your pages." — `docs/search/docs/crawling-indexing/site-move-no-url-changes.md`
- **[Site move / no URL change]** Remove temporary blocks before go-live: "Be sure to remove any such blocks from the new copy of the site when you're ready to start the move." — `docs/search/docs/crawling-indexing/site-move-no-url-changes.md`
- **[Site move / no URL change]** Lower DNS TTL ahead of the move: "Consider lowering the TTL to a conservative low value (for example, a few hours) at least a week in advance of the move to refresh DNS caches faster." — `docs/search/docs/crawling-indexing/site-move-no-url-changes.md`
- **[Removals]** Removals tool is temporary (~6 months): "Requests made in the Removals tool last for about 6 months." — `docs/search/docs/crawling-indexing/remove-information.md`
- **[Images]** robots.txt method targets `Googlebot-Image`: e.g. "User-agent: Googlebot-Image / Disallow: /images/dogs.jpg" — `docs/search/docs/crawling-indexing/prevent-images-on-your-page.md`
- **[Images]** Don't apply both block methods: "The two methods have the same effect ... Keep in mind that Googlebot has to crawl the URLs to extract the HTTP headers, so implementing both methods at the same time doesn't make sense." — `docs/search/docs/crawling-indexing/prevent-images-on-your-page.md`
- **[Images]** `noimageindex` is per-page, not per-image: "adding the `noimageindex` robots tag to a particular page will also prevent that images embedded in that page from getting indexed. However, if the same images also appear in other pages, they might get indexed through those pages." — `docs/search/docs/crawling-indexing/prevent-images-on-your-page.md`
- **[Share control]** Media indexing depends on crawl access: "Google only indexes images and videos that Googlebot is allowed to crawl." — `docs/search/docs/crawling-indexing/control-what-you-share.md`
- **[Redacted]** Edit images before embedding + OCR risk: "Completely remove or obscure any text or other non-public parts of the image, as OCR systems may turn any image text seen into searchable text." — `docs/search/docs/crawling-indexing/keep-redacted-information-out.md`
- **[Redacted]** Republish under a new URL after a leak: "Host the properly redacted document under a different URL." — `docs/search/docs/crawling-indexing/keep-redacted-information-out.md`
- **[Canonical-fix]** Verify Google's chosen canonical: "Use the URL Inspection tool to check which page Google considers canonical." — `docs/search/docs/crawling-indexing/canonicalization-troubleshooting.md`
- **[Canonical-fix]** For syndication, partners should block indexing: "The most effective solution is for partners to block indexing of your content." — `docs/search/docs/crawling-indexing/canonicalization-troubleshooting.md`
- **[Outbound]** `sponsored` for paid/ad links: "Mark links that are advertisements or paid placements (commonly called paid links) with the `sponsored` value." — `docs/search/docs/crawling-indexing/qualify-outbound-links.md`
- **[Outbound]** `ugc` for user-generated links: "We recommend marking user-generated content (UGC) links, such as comments and forum posts, with the `ugc` value." — `docs/search/docs/crawling-indexing/qualify-outbound-links.md`
- **[Outbound]** `nofollow` as the catch-all: "Use the `nofollow` value when other values don't apply, and you'd rather Google not associate your site with, or crawl the linked page from, your site. For links within your own site, use the robots.txt `disallow` rule." — `docs/search/docs/crawling-indexing/qualify-outbound-links.md`
- **[Outbound]** Regular links need no `rel`: "For regular links that you expect Google to fetch and parse without any qualifications, you don't need to add a `rel` attribute." — `docs/search/docs/crawling-indexing/qualify-outbound-links.md`
- **[Outbound]** `sponsored` now preferred over `nofollow` for paid links: "The `nofollow` attribute was previously recommended for these types of links and is still an acceptable way to flag them, though `sponsored` is preferred." — `docs/search/docs/crawling-indexing/qualify-outbound-links.md`
- **[AMP]** Don't let robots.txt block AMP: "Make sure that your robots.txt file doesn't block your AMP page." — `docs/search/docs/crawling-indexing/amp/enhance-amp.md`
- **[AMP]** Remove-only-AMP flow — drop the amphtml link and redirect: "Remove the `rel="amphtml"` link from the canonical non-AMP page in the source code." and "Configure your server to return either an `HTTP 301 Moved Permanently` or `302 Found` response for the removed AMP page." — `docs/search/docs/crawling-indexing/amp/remove-amp.md`
- **[AMP]** When AMP won't appear, unblock crawling: "Remove all robots `meta` tags and `X-Robots-Tag` HTTP headers from your canonical and AMP content." — `docs/search/docs/crawling-indexing/amp/validate-amp.md`

## Ignored / myths

- **[Redacted]** Visual "hiding" does not redact: "Even though putting text in a tiny font, using a font color that's the same as the background the text is on, or covering text with an image may make something invisible to the human eye, these methods don't actually redact material in a way that prevents search engines from indexing it and making it findable." — `docs/search/docs/crawling-indexing/keep-redacted-information-out.md`
- **[Redacted]** Black rectangles are not redaction: "avoid placing black rectangles over text as a redaction method, as this can result in the text still being included in the public document." — `docs/search/docs/crawling-indexing/keep-redacted-information-out.md`
- **[Redacted]** robots.txt is not a redaction tool — URLs still surface: "Even if a part of a website is blocked by robots.txt, the URLs may be indexed in search (without their content). Use hashes in URL parameters instead of email addresses or names." — `docs/search/docs/crawling-indexing/keep-redacted-information-out.md`
- **[Canonical-fix]** Declaring a canonical is not binding: "Even if you explicitly designate a canonical page, Google might choose a different canonical for various reasons, such as the quality of the content." — `docs/search/docs/crawling-indexing/canonicalization-troubleshooting.md`
- **[Outbound]** `rel` values are not a hard block on discovery/crawl: "Links marked with these `rel` attributes will generally not be followed. Remember that the linked pages may be found through other means, such as sitemaps or links from other sites, and thus they may still be crawled." — `docs/search/docs/crawling-indexing/qualify-outbound-links.md` — NOTE: the requested framing "nofollow is now a hint" is UNKNOWN — not stated in corpus; this corpus never uses the word "hint" for `nofollow`. It states only that such links "will generally not be followed" and "may still be crawled."
- **[AMP]** AMP gives no desktop search perk: "AMP on desktop doesn't get search-specific features in Google Search results." — `docs/search/docs/crawling-indexing/amp.md`

## Auditable checks

1. `[auto]` **AMP rel pairing** — for any page with `<link rel="amphtml" href=X>`, fetch X and confirm it carries `<link rel="canonical">` back to a canonical page, and that the canonical/non-AMP page points to the AMP via `rel="amphtml"`. Maps to AMP rel-pairing requirement (`validate-amp.md`, `enhance-amp.md`).
2. `[auto]` **AMP crawlable** — the AMP URL is not `disallow`ed for Googlebot in robots.txt, and (for pages meant to appear) carries no `noindex` `meta`/`X-Robots-Tag`. Maps to "your robots.txt file doesn't block your AMP page" (`enhance-amp.md`, `validate-amp.md`).
3. `[auto]` **A/B redirect status** — a redirect from an original URL to a variation URL returns `302` (temporary), not `301` (permanent). Flag `301` used for a variation redirect. (Distinguishing a genuine experiment from a real move is `[handoff]`.) Maps to website-testing 302-not-301.
4. `[auto]` **Cloaking probe** — fetch the same URL with a Googlebot user-agent and with a normal browser user-agent; flag materially different HTML/URL sets. Maps to "Don't cloak" (`website-testing.md`).
5. `[auto]` **Paused-site status codes** — if a site is intentionally paused, whole-site disable returns `503` (short outage) or an indexable `200` placeholder (longer); flag use of `403`/`404`/`410` or `noindex` to pause. Maps to pause status-code rules + prohibitions.
6. `[auto]` **robots.txt survives outage** — `GET /robots.txt` returns `200` (not `503`) and does not `Disallow: /` for all crawling. Maps to pause robots.txt caveats.
7. `[auto]` **Site-move redirect codes + chain length** — old→new URL redirects use permanent `301`/`308` (not `302`/`303`/`307`); redirect chains resolve directly, ideally ≤3 hops and fewer than 5. Maps to site-move recommended codes + avoid-chaining.
8. `[auto]` **robots.txt-as-block anti-pattern** — flag URLs that are `disallow`ed in robots.txt but appear intended for removal/deindexing (i.e., relying on robots.txt instead of `noindex`/password/removal). Maps to "Don't use robots.txt as a way to block your page" (`remove-information.md`).
9. `[auto]` **Image noindex is readable** — for images meant to be de-indexed via `X-Robots-Tag: noindex`, confirm the image URL is crawlable (not robots.txt-disallowed) so Googlebot can read the header. Maps to prevent-images "you must allow crawling."
10. `[auto]` **Image block-method exclusivity** — flag images that are BOTH robots.txt-disallowed AND serve `noindex` `X-Robots-Tag` (contradictory — the header can't be read when disallowed). Maps to "implementing both methods at the same time doesn't make sense."
11. `[auto]` **Outbound `rel` qualification** — parse crawlable `<a>` tags; verify ad/paid links carry `rel="sponsored"` (or legacy `nofollow`), UGC/comment links carry `rel="ugc"`, and that `rel` tokens are valid (`sponsored`/`ugc`/`nofollow`, space- or comma-separated). Maps to qualify-outbound-links taxonomy.
12. `[auto]` **Test hostname noindex** — staging/temporary hostnames (e.g., `beta.*`) serve a `noindex` rule. Maps to site-move-no-url-changes test-host guidance.
13. `[render]` **JS-driven variations / redirects** — render the page and detect client-side variation injection or `location`-based JS redirects used for A/B testing. Maps to website-testing "inserting variations dynamically ... use JavaScript to decide which variation to display"
14. `[handoff]` **Canonical mismatch** — use the Search Console URL Inspection tool to compare Google-selected canonical vs the declared canonical. Maps to canonicalization-troubleshooting.
15. `[handoff]` **Redaction leak inspection** — inspect published PDFs/images for change history, retained uncropped/OCR-able content, hidden metadata, or PII in URLs/filenames. Maps to keep-redacted best practices.
16. `[handoff]` **Removals-tool durability** — any URL submitted to the Removals tool is also backed by a permanent method (content removal, password, or `noindex`), since requests expire (~6 months). Maps to remove-information "last for about 6 months."
